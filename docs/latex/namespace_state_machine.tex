\doxysubsection{Detailed Description}
Namespace containing all typedefs, structures, and primary functions to implement and run your state machine. 

Basic API for implementing a Finite State Machine in your code.~\newline
 1) Edit NUM\+\_\+\+STT\+\_\+\+STATES macro to match the number of states in your code.~\newline
 2) Define and declare an enum of state types in a separate file if desired to improve readability of code.~\newline
 3) Define and declare your state machine and all state action functions. Each state action should return \mbox{\hyperlink{namespace_state_machine_aa4949a03e133bb180b63946bccb2e6dc}{State\+Machine\+::\+STT\+\_\+\+STATE}} and have 0 parameters.~\newline
 4) Add super state functionality to your state action functions by calling \mbox{\hyperlink{namespace_state_machine_a46656c81784d65a374ce5303590b8e5e}{State\+Machine\+::\+Process\+Super\+State()}} at the end of each sub-\/state. You should define and declare your super state action functions.~\newline
 5) In your superloop, call \mbox{\hyperlink{namespace_state_machine_ad07e38b551d396f767f3f6ae245df441}{State\+Machine\+::\+Execute\+Action()}} and pass the address of your state machine (i.\+e. State\+Machine\+::\+Execute\+Action(\&my\+\_\+state\+\_\+machine);).
